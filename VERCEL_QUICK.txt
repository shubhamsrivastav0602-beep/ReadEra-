## 🎓 FAQ

**Q: Will my books stay on Vercel?**  
A: Yes! books_data/index.json is deployed with the app.

**Q: How do I update books?**  
A: Update books_data/index.json → push to GitHub → auto-deploys.

**Q: What if I want to change admin password?**  
A: Update in Vercel environment variables → redeploy.

**Q: Can I use a custom domain?**  
A: Yes! Add in Vercel settings → point DNS → done.

**Q: Will Google Books cover images work?**  
A: Once you enable Books API in Google Cloud, yes. Until then, gradient covers work great.

**Q: What's the cost?**  
A: Free! Vercel free tier covers this easily.

**Q: Can I add users/login?**  
A: Yes, integrate Supabase (already configured).

**Q: Where I should push the code - do I need GitHub repo?**  
A: For easy deployment yes, but you can also use Vercel CLI directly.

## 📞 TROUBLESHOOTING

**Deploy fails?**
- Check all required files exist
- Verify requirements.txt has all dependencies
- Check vercel.json syntax

**API returns 500?**
- Verify environment variables inVercel settings
- Check books_data/index.json uploaded
- View Vercel function logs

**Links not working?**
- Clear browser cache
- Verify vercel.json routes
- Check file paths are relative

**Google Books not loading?**
- Enable Books API in Google Cloud Console
- Add API key to Vercel environment variables
- Gradient covers work as fallback
